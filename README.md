## commit

在 commit 代码时，commit message 请遵循以下格式

``` 
<Commit 类型>: <commit 描述>
```

commit 类型可以是：
* feat: 添加一个新的 组件/页面 或者为某个 组件/页面 添加了 feature
* docs：文档（documentation）
* refactor：重构（即不是新增功能，也不是修改bug的代码变动）
* test：本次 commit 修改了 test 中的某些内容
* chore：构建过程或辅助工具的变动
* utils: 修改了/src/utils 中的某些内容
* view: 视图修改
* bugs: 修改某个bug
